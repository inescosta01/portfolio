# Prompt

## Initial prompt

### Version 1

```text
Here, I want you to create a simple portfolio website, to a person that does digital marketing. Create a plan for it. 

This is to deploy in github pages as a static webpage, so no backend integrations or anything, I just want this to be publically available.

The person's name is 'Inês Costa'. All her professional information and personality is describes in the 'inces-costa-cv-pdf' I added at the root of the folder. 

Create a webpage that can tramit here creative work thorugh a not super hyper complex page but complex enough where you can feel here creativity. And do this based on themes and transitions that suit her personality
```

### Version 2

```markdown
Create a simple portfolio website for a digital marketing professional named **Inês Costa**.

**Steps:**
1. First, create a plan for the site structure and design direction before building anything.
2. Read `inces-costa-cv-pdf` (located at the root of the project folder) to extract her professional background, skills, experience, and personality traits. Use this to inform both the content and the tone/style of the site.

**Requirements:**
- Static site only — this will be deployed on GitHub Pages, so no backend, databases, or server-side integrations.
- The goal is to showcase her creative work and professional profile in a way that feels personal and expressive, not generic.
- Keep the design reasonably simple/lightweight, but with enough thoughtful detail (layout, theme, transitions, micro-interactions) that her creativity comes through — avoid both a bare-bones template feel and unnecessary over-engineering.
- Choose a color palette, typography, and transition/animation style that reflect her personality as described in the CV — justify these choices briefly in the plan.
- Ensure the final result is publicly accessible once deployed (correct relative paths, valid GitHub Pages config, etc.).

**Deliverable order:** plan first (structure, sections, theme rationale) → then implementation.
````
